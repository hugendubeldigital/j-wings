wingS FAQ

  1. What is wingS?
    wingS is net generation Swing. It is build on top of the swing libraries of the jdk1.2. I.e. using swing models, swing events and a very similar api. Mainly the class names and packages differ ;-). So if you know how to develop swing apps, you are able to develop wings apps.

  2. Why should I use wingS?
    Powerful, but easy to use.

  3. What are the requirements for wingS?
    Several things:
    • A computer :-)
    • Java 2
    • An application server capable handling serlvets.
      We can recommend Tomcat.
    • Unix/Linux: access to an X Server.

  5. How do I install the demos?
    This is easy if you have a state of the art application server which can handle web archives. Then you just have to deploy the wingset.war:
    • If you are using Tomcat, simply copy wingset.war into the webapps directory and restart Tomcat.
    • If you are using JBoss, simply copy wingset.war into the deploy directory. No restart is necessary.
    Then point your browser to http://yourserver.yourdomain/wingset.
    If you use some older stuff, I recommend downloading Tomcat or JBoss bundled with Tomcat ;-)

  6. Why does wingS require access to an X Server under Unix/Linux?
    We are using some AWT classes for the icons which enables you to use the methods you already know for images manipulation. Unfortunatly as soon as you are using AWT classes, the JVM wants to talk to an X Server because the AWT uses the system libraries for drawing.
    If you are using GNU/Linux, you can use i.e. Xvfb.
    If you want a quick and insecure sollution try "xhost +localhost" or "xhost +".

  7. Why does wingS use cookies?
    wingS itself does not use cookies, but (some) application servers use cookies to track the session. But you can normally configure them to not use cookies but url-rewriting if you like.

  8. How can I contribute?
    Please send your patches / fixes / extensions to wings-developers@wings.to.com. If you like subscribe to our mailing lists. You can also get write access to our cvs repository. Please visit http://www.wings.to.com/ for more information!

  9. How do I create a new servlet using wingS?
    This depends on the type of servlet you want to create:
    • If you only want a very simple one, you only need one class FooBarServlet which extends org.wings.servlet.SessionServlet and overwrites the method postInit to do something.
      See demo/wingset/Test.java for an example.
    • Otherwise you will need at least two classes, FooBar and FooBarSession.
      • FooBar should extend org.wings.servlet.WingServlet and overwrite the method generateSessionServlet which returns new instances of FooBarSession.
      • FooBarSession should extend org.wings.servlet.WingServlet and overwrite the method postInit to do something.
      See demo/wingset/WingSet.java and demo/wingset/WingSetSession.java for an axample.

  11. What is a plaf?
    wingS supports some kind of pluggable look and feel. Each SComponent delegates it's codegeneration to a special ComponentCG. There are different CGs for each SComponent and each plaf. For now, there exist two plafs xhtml/css1 and xhtml/old. Feel free to implement additional plafs like wap or xml/xsl ;-)
