wingS FAQ

  1. What is wingS?
    wingS is net generation Swing. It is build on top of the swing libraries of the jdk1.2. I.e. using swing models, swing events and a very similar api. Mainly the class names and packages differ ;-) (JTable pendant is STable, JTree pendant it STree,...). So if you know how to develop swing apps, you are able to develop wingS apps.

  2. Why should I use wingS?
    Powerful, but easy to use.

  3. What are the requirements for wingS?

    for compiling wingS

    • Java 2 with logging API
      We are using wingS with Sun's JDK 1.4. If you are using another Java 2 implemenation without the logging API, you could use the logging.jar in the lib directory of the wingS distribution.
    • A copy of JSDK2.2 API jar. You will find it in every servlet container implementing the JSDK2.2 API or on Sun's JSDK website. Copy it to the lib directory of your wingS installation.
    • Ant version 1.3 or higher.

    for running wingS applications

    • jsdk2.2 capable servlet container:
      • tomcat version 3.2 or higher
      • jetty
      • jboss with jetty/tomcat

  5. Why does wingS require access to an X Server under Unix/Linux?
    We are using some AWT classes for the icons which enables you to use the methods you already know for images manipulation and which loads images from file/classpath and calculates the size of images. Unfortunatly as soon as you are using AWT classes, the JVM wants to talk to an X Server because the AWT uses the system libraries for drawing.
    If you are using GNU/Linux, you can use i.e. Xvfb.
    If you want a quick and insecure solution try "xhost +".

  7. How do I create a new servlet using wingS?
    This is very simple: 
     import org.wings.*;

 public class HelloWorld {
   public HelloWorld() {
     SFrame frame = new SFrame();
     frame.getContentPane().add(new SLabel("Hello World"));
     frame.show();
   }
 }
    That's all. To integrate it in a servlet container you need to build a web archive (war file). How to do this, please look at the dokumentation of your servlet container. You must provide a web.xml file with similar content: 
    
<?xml version="1.0" encoding="ISO-8859-1"?>

<!DOCTYPE web-app
    PUBLIC "-//Sun Microsystems, Inc.//DTD Web Application 2.2//EN"
    "http://java.sun.com/j2ee/dtds/web-app_2.2.dtd">

<web-app>
  <servlet>
    <servlet-name>
      HelloWorld
    </servlet-name>
    <servlet-class>
      org.wings.session.WingServlet
    </servlet-class>
    <init-param>
      <param-name>wings.mainclass</param-name>
      <param-value>HelloWorld</param-value>
    </init-param>
    <init-param>
      <param-name>wings.lookandfeel.deploy</param-name>
      <param-value>/css1.jar</param-value>
    </init-param>
    <init-param>
      <param-name>wings.lookandfeel.name</param-name>
      <param-value>xhtml/css1</param-value>
    </init-param>
  </servlet>

<!-- url mapping -->

  <servlet-mapping>
    <servlet-name>
      HelloWorld
    </servlet-name>
    <url-pattern>
      /HelloWorld/*
    </url-pattern>
  </servlet-mapping>

</web-app>

    For more information please look at the online tutorial, which is a part of wingS (in the docs/tutorial dir) and implemented with wingS.

  9. What is a plaf?
    wingS supports some kind of pluggable look and feel. Each SComponent delegates it's codegeneration to a special ComponentCG. There are different CGs for each SComponent and each plaf. For now, there exist two plafs xhtml/css1 and xhtml/old. Feel free to implement additional plafs like wap or xml/xsl ;-)

  17. Filechooser upload isn't working: java.io.IOException: Datei oder Verzeichnis nicht gefunden
    You are using Tomcat 4.0.2 or 4.0.3 (or maybe an other version). Tomcat sets the system temporary directory to the "temp" directory in the tomcat directory, which does not exist. You can change this "strange" behaviour by set the environment variable CATALINA_TMPDIR to a directory of your choice, or create the "temp" directory in your tomcat installation dir.

  18. Why is SOptionPane not blocking on show... methods
    wingS is a single thread application. Each request get it's own thread to process the request. If a method is blocking, the thread blocks and wingS is not able to deliver the result of the request. There is no way around that

  19. Howto use SOptionPane
    SOptionPane did not block on show... methods, instead you have to do stuff depending on the result of the show method in an java.awt.ActionListener. An Example: 
    SOptionPane.showYesNoDialog(null, "do you catch it?", "Question?",
  new  ActionListener() {
    public void actionPerformed(ActionEvent e) {
      if ( e.getActionCommand()==SOptionPane.NO_ACTION ) {
        SOptionPane.showMessageDialog(null, "please read explanation again");
      } else {
        SOptionPane.showMessageDialog(null, "you are smart ;-)");
      }
    }
  });

  20. How can I access the session object from an other thread.
    If you start a thread, add this thread to the SessionManager with 
    org.wing.session.SessionManager.setSession(session);
    where session is the session of the parent thread. After that have access to the session in the new thread with 
    org.wing.session.SessionManager.getSession();
    At the end of the thread you should remove the session object from the sessionmanager with 
    org.wing.session.SessionManager.removeSession();
