Dynamic Resources

Introduction

WingS handles GUI components, that are rendered via a GUI description language (HTML, Stylesheets) that is transferred to a client (browser). The server and the client are typically separated by a network link with finite speed, therefore the communication between the client and the server should be minimized.

The following techniques are commonly used to address this problem:

  • Server: Send only changes.
    The server must determine, if and what has changed since the last dowload of the client and send only changes.
    Example: If an web-application has more than one HTML-frame, it makes sense to send only frames to the client whose content actually have changed.
  • Client: Caching.
    The client gets information from the server about the timeframe of validity of a resource, and stores it locally for that timeframe.
    Example: An icon, that is used in several buttons or labels, and does not change, can be cached for the entire lifetime of the session.
  • And, of course, minimize the size of the pages to be sent in the parts that can be influenced (like length of unique identifiers etc.)

WingS tries to maximize the transfer speed by utilizing all of the techniques above.
